# ShelfFlow 联调验收手册

## 前置条件

- 已安装 Docker Desktop，或本机已手动启动 MySQL / Redis / RabbitMQ。
- 默认后台账号来自初始化 SQL：`admin / 123456`。
- 已安装 Node.js 24+ 与 npm 11+。
- 已复制 `/Users/coconut/Desktop/ShelfFlow/.env.example` 到 `.env.local`，并填入真实 MySQL / Redis 连接信息。

## 启动顺序

1. 启动基础设施。如果本机已经有可用 MySQL / Redis / RabbitMQ，可以跳过这一步：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/start-infra.sh
```

2. 启动整条本地链路：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/start-local-all.sh
```

如果要把用户端 Web 一起拉起：

```bash
cd /Users/coconut/Desktop/ShelfFlow
START_USER_WEB=true bash scripts/start-local-all.sh
```

如果希望完整链路自动拉起基础设施：

```bash
cd /Users/coconut/Desktop/ShelfFlow
START_INFRA=true START_USER_WEB=true bash scripts/start-local-all.sh
```

3. 如需单独调试管理端 Web：

```bash
cd /Users/coconut/Desktop/ShelfFlow
npm install
bash scripts/dev-admin.sh
```

4. 如需单独调试用户端 Web：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/dev-user-web.sh
```

停止本地链路：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/stop-local-all.sh
```

停止 Docker 基础设施：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/stop-infra.sh
```

## 启动后检查

- Gateway：`${SHELFFLOW_GATEWAY_BASE_URL}/health`
- 管理端：`${SHELFFLOW_ADMIN_BASE_URL}`
- 用户端：`${SHELFFLOW_USER_WEB_BASE_URL}/products`
- MySQL / Redis / RabbitMQ：`scripts/start-infra.sh` 会等待 Docker healthcheck 就绪。
- 管理端登录接口可返回统一契约 `code: "ok"`

## 微服务 Smoke

脚本位置：

```text
/Users/coconut/Desktop/ShelfFlow/scripts/backend-smoke.sh
```

默认连接 `${SHELFFLOW_GATEWAY_BASE_URL}`，默认账号 `admin / 123456`：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/backend-smoke.sh
```

指定后台账号：

```bash
ADMIN_USERNAME=admin ADMIN_PASSWORD=123456 bash scripts/backend-smoke.sh
```

Smoke 覆盖接口：

- `GET /health`
- `POST /api/admin/auth/login`
- `GET /api/admin/auth/me`
- `GET /api/admin/products`
- `GET /api/admin/inventory-batches`
- `POST /api/admin/products`
- `PUT /api/admin/products/{id}`
- `POST /api/admin/inventory-batches`
- `PUT /api/admin/inventory-batches/{id}`
- `POST /api/admin/inventory-batches/{id}/status`
- `GET /api/admin/orders`
- `GET /api/admin/orders/{id}`
- `POST /api/admin/orders/{id}/status`

通过标准：脚本输出 `微服务 smoke 通过`，且每个接口均返回 HTTP 2xx 与统一响应 `code: "ok"`。

## 管理端功能验收

入口：

- `${SHELFFLOW_ADMIN_BASE_URL}`

建议按以下顺序验收：

1. 登录管理端，确认账号密码校验、失败提示与成功跳转正常。
2. 进入批次管理，确认批次列表可分页加载。
3. 创建商品，确认成功后重新拉取商品数据。
4. 创建批次，确认成功后重新拉取批次数据。
5. 编辑商品和批次，确认保存后列表立即刷新。
6. 修改批次状态，确认状态变化立即反映到列表。
7. 刷新页面，确认筛选条件、分页和数据状态一致。
8. 进入订单履约页，确认订单列表、详情和状态推进正常。

## 跨端订单闭环 Smoke

脚本位置：

```text
/Users/coconut/Desktop/ShelfFlow/scripts/order-lifecycle-smoke.sh
```

默认连接 `${SHELFFLOW_GATEWAY_BASE_URL}`，执行链路：

1. 用户登录
2. 用户加购并下单
3. 用户支付订单
4. 管理端查询订单
5. 管理端推进 `to_prepare -> preparing -> ready_for_pickup -> completed`
6. 用户侧确认订单最终状态为 `completed`

执行方式：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/order-lifecycle-smoke.sh
```

完整本地链路验收：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/admin-e2e.sh
```

现在 `admin-e2e.sh` 会依次启动 `legacy backend -> auth-service -> admin-service -> user-service -> gateway -> admin web`，然后执行：

- `backend-smoke.sh`
- `order-lifecycle-smoke.sh`

如果需要在冒烟通过后继续保留服务做浏览器联调，可使用：

```bash
cd /Users/coconut/Desktop/ShelfFlow
KEEP_SERVICES_RUNNING=true START_USER_WEB=true bash scripts/admin-e2e.sh
```

如果希望把用户端浏览器级 UI smoke 一起纳入整体验收，可使用：

```bash
cd /Users/coconut/Desktop/ShelfFlow
KEEP_SERVICES_RUNNING=true START_USER_WEB=true RUN_USER_WEB_UI_SMOKE=true bash scripts/admin-e2e.sh
```

这会额外启动用户端 Web，并在验收完成后保留服务，便于继续访问：

- `${SHELFFLOW_ADMIN_BASE_URL}`
- `${SHELFFLOW_USER_WEB_BASE_URL}`

在这套环境保留后，可继续执行用户端浏览器级 UI 冒烟：

```bash
cd /Users/coconut/Desktop/ShelfFlow
bash scripts/user-web-ui-smoke.sh
```

该脚本会驱动真实浏览器走以下链路：

1. 注册新账号
2. 找回密码并重置
3. 使用新密码登录
4. 商品目录进入详情页
5. 商品加购
6. 购物车数量调整
7. 提交订单
8. 支付订单
9. 查看订单列表

默认读取：

- `SHELFFLOW_USER_SMOKE_OPEN_ID`
- `SHELFFLOW_USER_SMOKE_NAME`
- `SHELFFLOW_USER_SMOKE_PHONE`
- `SHELFFLOW_USER_SMOKE_PASSWORD`

如果当前环境没有可售商品，脚本会明确失败并提示先执行 `admin-e2e.sh`。

## 常见问题

- `401`：先确认 `POST /api/admin/auth/login` 成功，且后续请求带上 `Authorization: Bearer <token>`。
- `Connection refused`：网关或下游服务未启动，或 `SHELFFLOW_GATEWAY_BASE_URL` 配置错误。
- Docker 基础设施未启动：执行 `START_INFRA=true bash scripts/start-local-all.sh`，或单独执行 `bash scripts/start-infra.sh`。
- 数据库连接失败：确认 `.env.local` 中 `DB_HOST DB_PORT DB_NAME DB_USER DB_PASSWORD` 配置正确。
- Redis 连接失败：确认 `.env.local` 中 `REDIS_HOST REDIS_PORT REDIS_PASSWORD REDIS_DATABASE` 配置正确，legacy backend 启动日志不应出现 Redis 连接异常。
